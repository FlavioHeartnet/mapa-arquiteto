# 00 — Roadmap: a espinha do mapa

O mapa é uma **corrida** (metáfora de pista): sai da **Largada**, passa por marcos (troféus) e cruza a **linha de chegada** já como Arquiteto de Soluções. A espinha principal é vertical; de cada nó saem ramos com sub-tópicos.

```
🏁 LARGADA
   │
   ├─ FASE JÚNIOR ────────────────────────────────
   │   Fundamentos → Linguagem → SQL → Versionamento
   │   → Redes/Web → APIs → Segurança → MVC → Docker
   │
   🏆 Programador Júnior
   │
   ├─ FASE PLENO / SÉNIOR ────────────────────────
   │   Bancos NoSQL → Clean Code → OO Avançada/SOLID
   │   → Object Calisthenics → Testes → Design Patterns
   │   → Refactoring → Performance → Cloud → CI/CD
   │
   🏆 Programador Completo!  →  Arquiteto de Software
   │
   ├─ FASE ARQUITETO ─────────────────────────────
   │   Arquitetura de Software (níveis/tipos/papéis)
   │   → DDD → Camadas → Monólito modular
   │   → Microsserviços → EDA → Serverless
   │
   🏆 Arquiteto de Software → Arquiteto de Soluções
   │   → Requisitos Não Funcionais (NFRs)
   │       ├─ Planejamento de Capacidade
   │       ├─ Escalabilidade e Performance
   │       ├─ Consistência de Dados
   │       ├─ Disponibilidade / Resiliência
   │       └─ Segurança como Requisito Arquitetural
   │   → AWS Certified → System Design
   │
🎽 PARABÉNS! (linha de chegada)
```

## As três fases (índices de "Tópicos" do board)

### Fase Júnior
Fundamentos · Escolha uma linguagem · SQL e Bancos relacionais · Versionamento de Código · Repositórios Remotos · Redes e Internet · Fundamentos da WEB · APIs & Integrações · Segurança · Model-View-Controller · Docker

### Fase Pleno / Sénior
Mais sobre bancos de dados · Clean Code Principles · Orientação a Objetos Avançada · Princípios SOLID · Object Calisthenics · Princípios de qualidade de Código · Pirâmide de Testes · Design Patterns · Técnicas de Refactoring · Otimização e Performance · Básico das Cloud Providers · Pipelines de CI/CD

### Fase Arquiteto
Arquitetura de Software · Níveis de Arquitetura · Tipos de Arquiteto · Papéis e Responsabilidades · Domain Driven Design · Modelagem Estratégica · Modelagem Tática · Arquitetura em Camadas · Monólito modular · Requisitos Não Funcionais (NFRs) · Planejamento de Capacidade · Escalabilidade e Performance · Consistência · Disponibilidade (HA) · Segurança como Requisito Arquitetural · Sistemas distribuídos e Padrões Arquiteturais

Detalhes de cada fase: [Júnior](01-fase-junior.md) · [Pleno/Sénior](02-fase-pleno-senior.md) · [Arquiteto](03-fase-arquiteto.md).
